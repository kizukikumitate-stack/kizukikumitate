# 記事テーマ提案スキル

## テーマの源泉

### 1. サイトのプログラム領域

きづきくみたて工房のプログラムから記事テーマを抽出する:

| プログラム | 記事テーマ例 |
|---|---|
| デモクラシーフィットネス | 民主主義の筋肉、対話と合意形成、学校での導入事例 |
| LEGO® SERIOUS PLAY® | 手で考える、メタファー、チームビルディング |
| 研修設計プログラム | 成人発達理論、HPI、適応課題、研修のプロとは |
| 組織開発プログラム | なぜ組織は変われないのか、システム思考、変革理論 |
| 対話力向上プログラム | 心理的安全性、組織的負債、対話のスキル |
| ソーシャルリーダーコーチング | 社会課題、リーダーの孤独、ブレイクスルー |

### 2. 過去の記事との差別化

note API で既存記事を取得して、重複しないテーマを選ぶ:

```bash
curl -s "https://note.com/api/v2/creators/kizukikumitate/contents?kind=note&page=1&per=20" | python3 -c "
import sys, json
d = json.load(sys.stdin)
for a in d['data']['contents']:
    print(f\"- {a.get('name','')}  ({a.get('publishAt','')[:10]})\")
"
```

### 3. トレンド・時事ネタとの掛け合わせ

時事ネタや季節のテーマと専門領域を掛け合わせる:
- 新年度（4月）→ 新チーム結成時の対話設計
- 年末（12月）→ 1年の組織変化を振り返る
- AI・テクノロジー → 生成AI × 研修設計
- 働き方改革 → リモートワーク × ファシリテーション

## 提案フォーマット

テーマを3つ提案する。各テーマに以下を含める:

```
### テーマ案1: [タイトル案]

- **切り口**: [なぜこのテーマが今読まれるか]
- **読者の課題**: [このテーマが刺さる読者の具体的な悩み]
- **構成案**:
  1. [導入: どんなシーンで始めるか]
  2. [本論ポイント1]
  3. [本論ポイント2]
  4. [本論ポイント3]
  5. [まとめ・読者への問いかけ]
- **想定文字数**: [2000〜4000文字]
- **関連プログラム**: [サイトのどのプログラムと連動するか]
```

## テーマ選定のルール

1. **読者の課題から逆算する** — 「書きたいこと」ではなく「読者が検索すること」から考える
2. **サイトのプログラムに誘導できる** — 記事を読んだ後「もっと知りたい」→ サイトのプログラムページへ
3. **過去記事と被らない** — 既存記事のタイトルと切り口を確認してから提案
4. **1記事1テーマ** — 詰め込みすぎない。1つの問いに1つの答え
