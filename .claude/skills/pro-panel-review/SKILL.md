# プロパネル・レビュー(8人の専門家による多視点レビュー)

きづきくみたて工房サイトのページを、8つの専門家視点で並行レビューし、
1本の優先度付きレポートに統合する。**提案のみ。ページの変更・コミットは一切しない。**

## パネルの構成(ペルソナ定義は `.claude/agents/panel-*.md` が唯一の正)

| エージェント | 視点 |
|---|---|
| `panel-info-designer` | 情報デザイン(深さのレイヤー・生きたデータ) |
| `panel-storyteller` | 物語(時間軸・航海日誌・プロローグとエピローグ) |
| `panel-game-ux` | 探索の楽しさ(進捗・隠し要素・参加の痕跡) |
| `panel-strategist` | セオリー・オブ・チェンジ(検証可能性・証拠) |
| `panel-workshop-designer` | 参加型デザイン(双方向性・対話の道具化) |
| `panel-creative-coder` | 実装品質(動きの質・a11y・パフォーマンス) |
| `panel-community` | コミュニティ(帰属感・支援導線) |
| `panel-educator` | 教育(教材性・ワークシート展開) |

## 手順

### 1. 対象ページを決める

- ユーザー指定があればそれに従う
- 指定がなければ: `git log --since="7 days ago" --name-only` で直近1週間に変更された
  公開ページ(ルート直下・yokai/・shakai-yokai/ の HTML。作業用 *-mockup.html 等は除く)から
  変更が最も大きい1〜2ページを選ぶ。変更がなければ rocket-map.html を対象にする
- 対象は最大2ページまで(レビューの質を保つため)

### 2. 素材を用意する

- 対象HTMLを読む(構造・文言・JS の把握)
- 可能なら Playwright でPC(1366px)とモバイル(390px)のスクショを撮って渡す
  (`.claude/skills/rocket-map/assets/shot.mjs` の手順を参照。撮れない環境ならHTMLのみで可)

### 3. 8人を並行で走らせる

サブエージェント(汎用エージェント)を8体、**1メッセージで並行に**起動する。各サブエージェントへの指示:

```
あなたはこれから「きづきくみたて工房」サイトのレビューパネルの一員を演じます。
まず .claude/agents/panel-<slug>.md を読み、その性格・見るポイント・出力形式・制約に
完全に従ってください。次に対象ページ <path> を読み、指定の出力形式でレビューを
返してください。提案のみで、ファイルは変更しないこと。
```

### 4. レポートに統合する

8人の出力を次の形式の Markdown レポート(panel-review-YYYYMMDD.md)にまとめる:

1. **対象と実施日** — ページ名・コミットSHA・実施日
2. **今週の最優先 TOP3** — 8人の提案から、効果と実装コストのバランスで3つ選ぶ。
   複数の専門家が同じ方向を指した提案を優先し、誰と誰が同意見かを明記する
3. **視点別フィードバック** — 8人それぞれの「良い点・提案・総評」を要約(各3〜5行)
4. **すぐやれる小ネタ** — 実装規模「小」の提案だけを一覧にする
5. **議論が分かれた点** — 専門家間で方向性が食い違った論点があれば正直に書く

### 5. 届ける

- レポートをユーザーに送付する(ファイルとして)
- 冒頭に1行で「今週のいちばんの提案」を書く

## 制約

- ❌ ページ・リポジトリへの変更、コミット、pushは絶対にしない(レビューは提案のみ)
- ❌ 8人の意見を平均化して丸めない。尖った意見は尖ったまま載せ、選別はTOP3で行う
- サイトの確定方針(文言トーン・数字ページの原則・フレームワーク禁止・台帳駆動)に
  反する提案には、レポート内でその旨を注記する
