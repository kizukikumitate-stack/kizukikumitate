# note下書き保存スキル

## 前提条件

- Chrome ブラウザで note.com にログイン済みであること
- Claude in Chrome MCP（または Control Chrome MCP）が利用可能であること

## 手順

### Step 1: noteの新規記事作成ページを開く

```
Chrome MCPで https://note.com/notes/new を開く
```

ページが読み込まれるまで2〜3秒待つ。

### Step 2: スクリーンショットで確認

noteのUIが変更されている可能性があるため、必ずスクリーンショットを撮って現在のUI構造を確認する。

確認ポイント:
- タイトル入力欄の位置
- 本文入力欄の位置
- 「下書き保存」ボタンの位置
- 「公開」ボタンの位置（**絶対に押さない**）

### Step 3: タイトルを入力

1. タイトル入力欄をクリック（通常はページ上部の大きなテキストエリア）
2. 記事タイトルをtype入力

### Step 4: 本文を入力

1. 本文入力欄をクリック（タイトルの下のエリア）
2. 本文を段落ごとに入力

**注意: noteのエディタはリッチテキストエディタ**
- 見出し（h2）は本文入力後に選択して見出しスタイルを適用
- まずはプレーンテキストで全文を入力し、後からフォーマットを調整する方法が安全
- 改行は Enter キーで行う

### Step 5: 下書き保存

1. スクリーンショットで「下書き保存」ボタンの位置を確認
2. 「下書き保存」ボタンをクリック
3. 保存完了のフィードバック（トースト通知等）を確認

**⚠ 絶対に「公開」「公開設定」ボタンは押さない**

### Step 6: 結果を報告

```
下書きに保存しました。
noteの下書き一覧（https://note.com/dashboard/drafts）から確認・編集できます。
内容を確認して、問題なければ手動で公開してください。
```

## トラブルシューティング

### noteにログインしていない場合

```
noteにログインされていないようです。
以下の手順でログインしてください:
1. Chromeで https://note.com/login を開く
2. アカウント情報でログイン
3. ログイン完了後、再度この操作を実行してください
```

### UIが変更されていてボタンが見つからない場合

Chrome MCPでの自動操作を中止し、手動コピペの手順をガイド:

```
noteのUIが変更されているため、自動入力ができませんでした。
以下の手順で手動で下書きを作成してください:

1. https://note.com/notes/new を開く
2. 以下のタイトルをコピーしてタイトル欄に貼り付け:
   [タイトル]
3. 以下の本文をコピーして本文欄に貼り付け:
   [本文]
4. 「下書き保存」をクリック
```

### 本文が長すぎてtype入力が遅い場合

クリップボード経由でペーストする:
1. 本文入力欄をクリック
2. JavaScriptでクリップボードにテキストを設定
3. Cmd+V でペースト

## 安全ルール

1. **公開ボタンは絶対に押さない** — 下書き保存のみ
2. **操作前にスクリーンショットを撮る** — UIの確認を怠らない
3. **入力後にスクリーンショットで結果を確認** — 正しく入力されたか目視確認
4. **ユーザーの承認なしに操作を開始しない** — 記事内容の承認後にのみ実行
