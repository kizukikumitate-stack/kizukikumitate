# アクセス解析の定例レビュー

計測基盤の詳細は `~/kizukikumitate-ops/analytics/looker-dashboard-setup.md`。
GA4プロパティ `G-2ECK6163X4`、**全75ページ計測済み（2026-07-20開始）**。

## 実行

```bash
cd /Users/morimotoyasuhito/kizukikumitate
node scripts/ga-report.mjs --days 28    # 月次
node scripts/ga-report.mjs --days 7     # 週次
```

- ★ネットワークがサンドボックス不許可のため **`dangerouslyDisableSandbox: true`** で実行する。
- 出力先: `~/kizukikumitate-ops/analytics/reports/YYYY-MM-DD-NNdays.md`
- 認証情報は `~/.config/kizukikumitate/ga4-service-account.json`（閲覧者権限のみ・リポジトリ外）。
  **中身を表示しない・コマンドラインに渡さない**。

## 週次（5分）— 健康診断であって、改善判断ではない

`--days 7` で見るのはこの3つだけ。

1. 全体の表示回数が**極端に落ちていないか**（落ちていればサイトかDNSの障害を疑う）
2. 告知した日にスパイクが出たか（メルマガ・SNSの効果）
3. `begin_checkout` が発生したか

**週次で率を語らないこと。** 分母が小さく、1人の増減で倍半分に振れる。

## 月次（30分）— ここで「次に直す1ヶ所」を決める

### 手順

1. `--days 28` でレポートを生成する
2. 前回のレポート（`reports/` の1つ前）と並べて読む
3. `~/kizukikumitate-ops/analytics/changelog.md` を見て、**前回何を変えたか**を確認する
4. 下の判断表に当てはめて、**変更点を1つだけ**提案する
5. 実行したら changelog に記録し、GA4のメモにも残す（後述）

### 判断表（有料導線）

分母が30未満なら「まだ判断しない」が正解。無理に動かすと、ノイズを追いかけることになる。

| 症状 | 解釈 | 次にやること |
|---|---|---|
| 計算機の表示が少ない | そもそも人が来ていない | 集客側。記事・メルマガ・SEO。LPを触っても無駄 |
| 表示はあるが送客率が低い | 計算機からキットへの動線が弱い | リンクの位置・文言・出るタイミング |
| 送客できるが押下率が低い | LPの中身が刺さっていない | 価格の見せ方・不安の解消・証拠の提示 |
| 押下率は高いが売れない | 決済画面で落ちている | Stripeの決済手段・価格・表示名 |

### 判断表（コンテンツ）

| 症状 | 次にやること |
|---|---|
| 特定の図鑑が上位に来ている | **当たり**。そのページに次の導線（関連図鑑・キット・体験会）を足す |
| 検索チャネルが伸びている | ストックが効き始めた。同系統のページを増やす価値がある |
| 直接・参照ばかり | 自分の告知に依存。放置すると止まる。SEOかSNSの種まきが必要 |
| 表示2回以下のページ | 一度告知して反応を見る → それでも動かなければ畳むか統合 |

## 記録（これが無いと因果が追えない）

### 1. changelog に書く

`~/kizukikumitate-ops/analytics/changelog.md` に1行追記する。

```markdown
- 2026-08-03 キットLPの価格提示を「¥2,700」から「1回の昇給で回収」に変更（押下率2.1%→?）
```

### 2. GA4のメモに残す

GA4 → 管理 → データの表示 → **メモ** で、同じ内容を日付つきで登録する。
グラフに縦線が入り、「いつ何を変えたら数字が動いたか」が後から見える。

## 原則

- **1サイクル1変更。** アクセスが少ないのでA/Bテストはできない。順番に1つずつ変えるしか原因を特定する方法がない。
- **提案は1つに絞る。** 「やるべきこと5選」を出さない。実行されない提案は無いのと同じ。
- **分母を必ず添える。** 「押下率8%」ではなく「押下率8%（LP表示25件中2件）」と書く。
  後者を見れば、判断に使えない数字だと自分で分かる。
- 森本さんは制作速度が速く、**作る方向には自然に進む**。このスキルの役目は
  「作る」ではなく「どれが効いたかを見て、次を絞る」こと。増やす提案に流れない。
